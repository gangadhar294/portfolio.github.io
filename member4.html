GreenWave Protocol

Zero-Latency Emergency Response System

The Business Problem

Our city's "Stadium Junction" causes 400% delays during match days. Ambulances carrying critical patients from the Industrial Zone to the City Hospital were getting stuck, risking lives.

Solution: A dynamic routing engine that calculates weights based on real-time traffic density, not just distance.

Algorithmic Logic

We model the city as a weighted graph where intersections are Nodes and roads are Edges. The weight $W$ is a function of distance ($d$) and congestion ($c$): $W = d \times (1 + c)$.

Since weights are non-negative, **Dijkstra’s Algorithm** is the optimal choice. It uses a Priority Queue to Greedily select the closest unvisited node, guaranteeing the mathematically shortest time.

HydroGrid Network

Infrastructure Cost Minimization Engine

The Business Problem

Connecting the remote "High Altitude Zone" to the water supply is capital intensive. Piping costs ₹10,000 per meter. We needed a blueprint to connect all residential zones to the central tank with the minimum possible pipe length.

Solution: Building a Minimum Spanning Tree (MST) that connects all $V$ nodes using exactly $V-1$ edges with minimum total weight.

Algorithmic Logic

We utilized **Prim’s Algorithm**. Unlike Kruskal’s (which sorts edges), Prim’s grows the tree from a starting source (Water Plant).

It maintains a Min-Heap of pipes connecting the 'Built' network to the 'Unbuilt' zones, always picking the cheapest connection next. This prevents cycles, ensuring consistent water pressure.

RapidExit System

Multi-Agent Disaster Evacuation Protocol

The Business Problem

The Port and Beach zones are Tsunami-prone. In a panic, standard GPS directs everyone to the main highway, causing fatal gridlock. We needed a system to find the nearest exit for thousands of people simultaneously.

Algorithmic Logic

We treat the city map as an **Unweighted Grid**. The goal is finding the path with the fewest "hops" (street crossings) to a Safe Zone.

**Breadth-First Search (BFS)** is perfect here. It explores the map layer-by-layer (like a ripple in water). It guarantees that the first time we reach a Safe Zone node, it is via the shortest path.

CargoMax Optimizer

Port Logistics & Yield Management

The Business Problem

Our Port storage has a fixed weight capacity. Ships arrive with varied containers: High-Profit/Low-Weight (Electronics) vs Low-Profit/High-Weight (Coal). We must select the mix that maximizes profit.

Why Greedy Fails: Picking just the most expensive item might waste space. Picking the lightest might not yield profit.

Algorithmic Logic

We use **0/1 Knapsack Dynamic Programming**. We build a 2D table `DP[item][capacity]`. At each step, we make a decision: "Do we include this container or not?" based on which choice yields a higher value for the remaining capacity.

SwiftLink Delivery

Last-Mile Logistics Optimization

The Business Problem

Delivery trucks leave the Warehouse, must drop parcels at University, Market, and Residential zones, and return. Random driving wastes fuel.

Algorithmic Logic

This is the **Traveling Salesman Problem (TSP)**. We need a Hamiltonian Cycle with minimum weight.

For small delivery clusters ($N < 20$), we use **Held-Karp Algorithm** (Dynamic Programming with Bitmasking). This provides the exact optimal route significantly faster than brute force ($O(N!)$).
