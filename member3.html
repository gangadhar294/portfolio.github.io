Smart Grid Street Lighting

Kruskal’s MST
The Business Case

We are upgrading the city's Energy Grid. The challenge is to connect 15 new neighborhood substations to the main power plant. Copper cabling is extremely expensive. We must find the layout that connects 100% of stations with the absolute minimum total cable length.

Graph Modeling

Nodes: Substations. Edges: Possible cable routes. Weights: Cost of laying cable ($/km). The result must be a Spanning Tree (no loops, full connectivity).

Algorithm Analysis

Kruskal’s Algorithm is ideal for sparse networks. It sorts all potential connections by cost and iteratively adds the cheapest ones, merging separate clusters of stations until the whole city is powered.

Implementation Logic
> SORT Edges by Cost > FOR each Edge(u, v): IF Find_Set(u) != Find_Set(v): Union(u, v) Add Cable to Grid ELSE: Discard (Redundant Loop)
Blueprint Analysis
TIME COMPLEXITY O(E log E)
SPACE COMPLEXITY O(V + E)

Metro Rail Fare System

Floyd-Warshall
The Business Case

The new City Metro has 20 stations. To generate the Automated Fare Chart, we need to know the shortest distance between *every* possible pair of stations (not just from a single source). This matrix allows the ticketing system to price rides accurately.

Graph Modeling

Nodes: Metro Stations. Edges: Track segments between adjacent stations. Goal: All-Pairs Shortest Path matrix.

Algorithm Analysis

Floyd-Warshall is a dynamic programming algorithm that solves the All-Pairs Shortest Path problem in O(V³). While slower than Dijkstra for a single path, it computes the entire network grid in one pass, perfect for static rail networks.

Implementation Logic
> INIT Matrix D[i][j] = Weight(i,j) > FOR k from 1 to N: FOR i from 1 to N: FOR j from 1 to N: D[i][j] = MIN( D[i][j], D[i][k] + D[k][j] ) > OUTPUT: Fare Matrix
Blueprint Analysis
TIME COMPLEXITY O(V³)
SPACE COMPLEXITY O(V²)

Fiber Network Resilience

DFS (Articulation Points)
The Business Case

To ensure 99.9% uptime for the city's internet, we must identify Critical Hubs (Single Points of Failure). These are specific router nodes that, if they fail, would split the city's network into disconnected islands.

Graph Modeling

Articulation Points: In a graph, an Articulation Point is a node whose removal increases the number of connected components. Finding these allows us to build redundant backup lines specifically where they are needed most.

Algorithm Analysis

We use Tarjan’s Algorithm (DFS based). It is highly efficient O(V+E). It tracks "Discovery Time" and "Low Link Value" for each node to detect back-edges and bridges.

Implementation Logic
> DFS(u): Children = 0 Disc[u] = Low[u] = ++Time FOR v in Adj[u]: IF v is Parent: Continue IF v Visited: Low[u] = Min(Low[u], Disc[v]) ELSE: DFS(v) Low[u] = Min(Low[u], Low[v]) IF Low[v] >= Disc[u]: ART_POINT = True
Blueprint Analysis
TIME COMPLEXITY O(V + E)
SPACE COMPLEXITY O(V)

Solar Energy Capacity

Fractional Knapsack
The Business Case

City Hall has a limited roof area of 500 sq meters. We have various solar panel types (Monocrystalline, Polycrystalline, Thin-Film) with different efficiencies and sizes. We need to fill the roof to maximize total energy output.

Graph Modeling

Greedy Optimization: Unlike cargo (0/1), solar panels can be cut or arranged to fill partial spaces. We calculate Value-Per-Square-Meter for each type.

Algorithm Analysis

Fractional Knapsack (Greedy Strategy). We sort items by efficiency (Value/Weight ratio) and take as much of the best panels as possible, then the next best, until the area is full.

Implementation Logic
> CALCULATE Ratio R = Energy / Area for all panels > SORT Panels by R (High to Low) > FOR each Panel P: IF Area_Left >= P.Area: Take Whole Panel ELSE: Take Fraction (Area_Left / P.Area) BREAK
Blueprint Analysis
TIME COMPLEXITY O(N log N)
SPACE COMPLEXITY O(1)

CCTV Data Compression

Huffman Coding
The Business Case

The city's 5,000 CCTV cameras generate Terabytes of footage daily. Storing raw data is too costly. We need a lossless compression algorithm to reduce storage size based on the frequency of pixel patterns (e.g., static backgrounds vs moving cars).

Graph Modeling

Binary Tree: We build a tree where frequently occurring data patterns (like a black night sky) are near the root (short codes), and rare patterns (an explosion) are deep leaves (long codes).

Algorithm Analysis

Huffman Coding is a greedy algorithm that builds an optimal prefix code. It ensures the most common data uses the fewest bits, resulting in 30-50% storage savings.

Implementation Logic
> COUNT frequency of all characters/patterns > CREATE leaf nodes for each char > INSERT into Priority Queue > WHILE Queue size > 1: Extract 2 min nodes (Left, Right) Create new internal node (Sum of freq) Insert back to Queue
Blueprint Analysis
TIME COMPLEXITY O(N log N)
SPACE COMPLEXITY O(N)
