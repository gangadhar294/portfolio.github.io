Logistic Transportation

Dijkstra Algorithm
The Business Case

In a city, logistics transportation includes activities like last-mile delivery, warehousing, freight movement, and distribution planning. Common modes are trucks, vans, two-wheelers, rail freight (where available), and increasingly electric vehicles. The main goals are to reduce traffic congestion, lower pollution, cut delivery time, and minimize costs.

Graph Modeling

Nodes (Vertices): - Warehouses, distribution centers, retail stores.
Edges:- Roads/rail routes between them.
Weights: - Distance, travel time, or cost.

Algorithm Analysis

Dijkstra's AlgorithmIt is widely used to find the shortest or least-cost path between a warehouse and delivery location in a transportation network, helping minimize travel time, distance, and fuel cost. optimizes multi-stop delivery routes.

Implementation Logic
>{ Input: n (number of locations), m (number of roads) For each road: add edge (u, v, cost) to G Input source (warehouse) For i = 0 to n-1: dist[i] = ∞ visited[i] = false dist[source] = 0 Repeat n times: u = node with minimum dist[] and not visited visited[u] = true For each neighbor v of u: if visited[v] == false AND dist[v] > dist[u] + cost(u, v): dist[v] = dist[u] + cost(u, v) Output dist[] }

Bank Management

Dijkstra's Algorithm
The Business Case

Managing thousands of customer accounts and transactions securely requires instant access and verification to prevent fraud and delays in the financial district.
Bank management in a city involves planning, organizing, and controlling banking operations to provide efficient financial services to citizens, businesses, and industries.
Nodes (Vertices) - Bank branches,ATMs,Customers,Central Server.
Edges-Transaction Links,Cash transfer routes,Communication skills.
Weights - Transaction time,Cost Or Network delay.

Algorithm Analysis

Hashing allows O(1) access to account details. RSA Encryption ensures secure transactions between the bank and vendors.
1. Time Complexity
Using adjacency list + priority queue:
O((V + E) log V)
where
V = number of branches/ATMs (nodes)
E = number of transaction links (edges)

2. Space Complexity
O(V + E)
Distance array → O(V)
Graph storage → O(E)

Implementation Logic
{ Input: n (number of branches/ATMs), m (number of links) add edge (u, v, time) to G Input source (main branch) For i = 0 to n-1: dist[i] = ∞ visited[i] = false dist[source] = 0 Repeat n times: u = unvisited node with minimum dist[u] visited[u] = true For each adjacent node v of u: if visited[v] == false AND dist[v] > dist[u] + time(u, v): dist[v] = dist[u] + time(u, v) Output dist[] }

City waste Management and Recycling

Travelling Salesman Problem(TSP)
The Business Case

Waste collection trucks must visit all locations once and return to the depot.
Exact TSP is NP-hard, so for large cities, heuristic or approximation algorithms are used:
->Nearest Neighbor
->Genetic Algorithm
->Ant Colony Optimization
->Simulated Annealing.

Algorithm Analysis

B-Trees or B+ Trees1. Time Complexity
Exact TSP: O(n!) → impractical for large cities (n = number of bins/locations)
Heuristic Algorithms:
Nearest Neighbor: O(n²)
Genetic Algorithm: O(g × n × p)
g = number of generations
n = number of locations
p = population size.
2. Space Complexity
Graph storage: O(n²) for adjacency matrix or O(n + E) for adjacency list
Additional structures (routes, pheromones, population) → O(n) to O(n × p). .

Implementation Logic
>{ Input: n (number of waste collection points) dist[n][n] (distance matrix between points) visited[n] = false route = empty list start = 0 current = start visited[current] = true For count = 1 to n-1: minDist = ∞ nextNode = -1 For i = 0 to n-1: if visited[i] == false AND dist[current][i] < minDist: minDist = dist[current][i] nextNode = i current = nextNode visited[current] = true add current to route Output route }

Disaster Evacuation

BFS
The Business Case

Safely move people from danger zones to the nearest safe shelters in minimum steps.
Nodes: Affected areas, intersections, shelters
Edges: Roads.

Algorithm Analysis

Merge Sort organizes listings efficiently. Collaborative Filtering1. Time Complexity
O(V + E)
V = number of locations (areas, intersections, shelters)
E = number of roads
BFS visits each node and edge once.
2. Space Complexity
O(V)
Queue for BFS
Visited array to mark reached locations

Implementation Logic
>{ Input: V (number of locations),E (number of roads) Graph[V] (adjacency list),source (danger zone) visited[V] = false distance[V] = -1 visited[source] = true distance[source] = 0 While Q is not empty: u = remove front element from Q For each v in Graph[u]: if visited[v] == false: visited[v] = true distance[v] = distance[u] + 1 if v is a safe shelter: Output distance[v] Stop Output distance[]
}

City Tax Records

AVL Tree
The Business Case

City tax data must remain sorted (by property ID / taxpayer ID).
Frequent updates (new taxpayers, deletions, corrections).
AVL Tree remains height-balanced, guaranteeing O(log n) time.

Node: Taxpayer ID, name, property ID, tax amount
Left Child: Smaller ID
Right Child: Larger ID
Operations->
Insert new tax record
Search existing taxpayer record
Delete obsolete or merged records
Update tax amount efficiently

Algorithm Analysis

Stack1. Time Complexity
Operation Complexity:

Search O(log n)
Insert O(log n)
Delete O(log n)
Rotation O(1)
AVL tree remains height-balanced, so tree height is O(log n).

2. Space Complexity
O(n) for storing n tax records
O(log n) auxiliary space (recursion stack during insertion/deletion)

Implementation Logic
>{ Structure Node: key (Taxpayer ID , Property ID) height,left, right Function Height(node): if node == NULL return 0 return node.height Function BalanceFactor(node): return Height(node.left) - Height(node.right) Function RightRotate(y): x = y.left T2 = x.right x.right = y y.left = T2 return x Function LeftRotate(x): y = x.right T2 = y.left y.left = x x.right = T2 return y Function Insert(node, key): if node == NULL: return node if key < node.key: node.left = Insert(node.left, key) else if key > node.key: node.right = Insert(node.right, key) else: return node update node.height balance = BalanceFactor(node) if balance > 1 AND key < node.left.key: return RightRotate(node) if balance < -1 AND key > node.right.key: return LeftRotate(node) if balance > 1 AND key > node.left.key: node.left = LeftRotate(node.left) return RightRotate(node) if balance < -1 AND key < node.right.key: node.right = RightRotate(node.right) return LeftRotate(node) return node Function Search(node, key): if node == NULL OR node.key == key: return node if key < node.key: return Search(node.left, key) else: return Search(node.right, key) Output: Efficient insertion and search of city tax records }
